Acquire data

Data acquisition operations are managed through the following view:

Interface element Function
Start /Stop button Press "Start" to begin an acquisition - the program will start acquiring data immediately or wait for an external event to occur (for example a TTL trigger). Press this button again to stop acquisition.
2 choices button:
Write to disk
Oscilloscope 		2 data acquisition modes are available:
- write to disk: save data to a file stored on a disk : if you select this option, before acquiring data, you will need to provide information about the experiment and the file where data should be stored.
- acquire data and display them on the screen. Data are not saved and cannot be used for anything else. This mode is useful when trying to impale a cell or checking the noise level of the system.
Board Data acquisition board to which the commands will be channeled and which will send data to the program.
This interface has been specifically designed to work with Data Translation cards running under a generic driver called DTOpenLayeres. If no card is present, this dropbox should be empty.
A/D parameters ---
Sampling mode... dialog box to adjust data acquisition parameters concerning the sampling rate and time
Input channels... dialog box to adjust data acquuisition parameters concerning the input channels.
D/A parameters --- (displayed only if the card has digital-to-analog (D/A) capabilities
Enable output check this option if D/A function is needed
Output mode... dialog box to select mode of operation
Output channels... dialog box to select which channels are used
   
Sweep length Length of data displayed on the screen. You can change this parameter through the "Sampling mode..." dialog box.
Ordinates Ordinates scale in mV
Abcissa Abcissa scale in s
[zoom][level] buttons
scrollbar 		The function of the scroll bar depends on which button is selected:
- zoom (default): enables to change the zoom level of the display (max to min ordinates)
- level: enables to change the span of the display, moving max and min together.

Notes:

Data are acquired as chunks of data, i.e. into consecutive arrays. Once an array has been filled by the A/D card, the card sends a message to the program which will then store the data on the disk and display it on a portion of the screen. If the card and the program cannot follow the sampling rate you need, try to enlarge the size of the buffers, reduce the number of buffers per sweep, and ultimately, disable the display.

Ordinates and abcissa information are tentative and have not been thoroughfully checked.

D/A operations are still under development.
