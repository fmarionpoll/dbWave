Define sampling settings

This dialog box let you define how signals are acquired (at which temporal resolution), what they will become, and how the data acquisition will start. It displays 3 types of settings.

A/D card parameters

This set of parameters defines how fast you will sample incoming data. Usually, we sample each channel at 10 kHz. It also defines how data are displayed on the screen as consecutive sweeps (here: 1 s), and also how data are stored in memory (here: as 10 buffers of 1000 words). These parameters are critical if your memory resources are limited and if the computing resources are limited. It is best to have only dbWave running while acquiring data.

Data flow

This defines mainly how long data will be stored when you decide to save them on the disk. "Continuous / limited" was meant to allow the user to interrupt the acquisition on demand, but this option is not currently set. "Duration" defines how long the data acquisition bout will be. Usually, we perform 2 s acquisition bouts for our taste recordings. "On-line" display determines if data are displayed on the screen while acquiring data. "Write to disk" defines if data are sent along to the disk or not. "Audible notification" defines if a "bip" is emitted at the end of the data acquisition.

Trigger

These parameters define how the acquisition will start. "Software" means that it will start immediately after defining the saving parameters, "Keyboard" means that the program will launch a dialog box waiting for the user to press a key, and "external trigger" means that the A/D box is connected to a wire which will deliver a TTL pulse to star the acquisition (A/D TRIG).

In this dialog box, make sure that "software trigger" or "keyboard trigger" are selected.

If you hit OK, you return to the previous screen, if you hit "Channels...", you will return to the amplifier settings dialog box.
