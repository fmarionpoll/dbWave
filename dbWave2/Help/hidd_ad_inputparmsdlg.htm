Define input parameters for acquisition

This dialog box let you adjust how your data will be acquired and how much amplification is present on each input channel:

In this example, we have decided to acquire data from 2 channels.
Interface element Function
Upper panel  
Nb. chans: n Number of data acqusition channels
Max.: 32 information: maximum number of data acquisition channels supported by the cards
A/D card  
Single ended / differential Defines how the input signal is connected to the card. In single-ended mode, each input channel is measured across a common grounds. In differential mode, each input channel is measured across another channel, which serves as a reference. This mode is supposed to be less prone to common mode noise.
Model: DT98000 information: A/D card currently connected.
Resolution (bits): 16 information: precision of the measure.
Encoding: offset binary information: type of encoding of the binary data.
   
Table Individual A/D channel settings (table rows 1 to 3): name, channel number and gain
A/D channels (0/7) Data acquisition card parameter: input channel (! numbers go from 0 to 7).
gain Gain of the data acquisition channel (available values: 1, 2, 4, 8)
Amplifier Text descriptor of the amplifier connected to the A/D card. If the amplifier is defined as "CyberAmp 320", then dbWave will directly send commands to change the settings of the amplifier, provided that the CyberAmp is connected to the computer via a serial cable. Information about the following rows (channel to low pass filter) are provided under the assumption that your computer is connected to a CyberAmp. If it were not the case, you need to report the correct values of these parameters manually and change these parameters directly on the amplifier (if you use one).
channel CyberAmp 320 have 2 channels (1 and 2) - any changes of the gain or filter settings of the rows in the same column will be sent to the Cyberamp corresponding channel.
gain total amplification gain
notch filter check if you wish to use CyberAmp 50/60 Hz notch filter
IN + filter (Hz) This is the input to which you generally connect your incoming signal. This input is positive which means that your signal is just amplified and filtered. Available values are GND, DC, 0.1, 1, 10, 30 , 100 Hz. If "GND" is selected, the input signal is blocked (nothing will come from the outside) - it provides a way to check for the noise sources which affect the transmission of the signal between the amplifier and the computer. DC means that the signal is not filtered.
IN- filter (Hz) This input is generally set to GND. Each CyberAmp channel is differential, which means that it has 2 inputs which are subtracted to build the final signal. The input to this channel is inverted. If you set this input to GND, then the output signal will be only the signal that was present at IN+
low pass filter (Hz) Set this filter to at least 1/2 of the sampling frequency to avoid phantom frequencies
Probe Name of the probe (amplifier connected to the biological preparation)
gain Gain of the probe
Signal max/min Information: signal range that can be handled by the A/D card. If the signal goes out of this range, it will be clipped. If you use a total gain of x10, the limits will be +1 V to -1 V. With a gain of x100, it will be +100 to -100 mV.
total gain Information: total gain applied, taking into account the gain of the A/D card, of the amplifier and of the probe.
resolution (µV) Information: voltage step (in microvolts) between consecutive numbers. For a card which converts voltage values into 16 bits numbers, it means that for a gain of 1, the voltage encoded by 1 unit is 20 V / 65536.
   
Buttons  
OK / Cancel OK will confirm and save the current settings
Channels... (greyed - it refers to the companion Dialog box where the same button is active)
Sampling... Jump to the companion dialog box which defines sampling intervals

Note:

These parameters will be saved when you exit the program. However, if you go to "database" and return to "Record", the program will load the data acquisition parameters of the record that was selected in the database and erase your current parameters.
