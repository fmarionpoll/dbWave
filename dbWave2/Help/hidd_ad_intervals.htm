Define time parameters for data acquisition

This dialog box let you define how signals are acquired (at which temporal resolution), what they will become, and how the data acquisition will start. It displays 3 types of settings.

Interface element Function
A/D card parameters This set of parameters defines how fast you will sample incoming data.  These parameters are critical if your memory resources are limited and if the computing resources are limited.
The 4 parameters in this box are related by the following formula:
sweep_len = n_words * n_buffers / rate
For the same sweep length, if you have small buffers, the screen will be refreshed quickly and by small increments; if you define large buffers, you will have less buffers and the screen will be refreshed less frequently. Typically, we use 1 s = 1000 words x 10 buffers / 10 000 Hz.
Rate/channel (Hz) Typical: 10 000 Hz (rate). Check the technical specifications of your A/D card. For many cards, the maximum sampling rate is given as the total number of data points that can be acquired per second. If you acquire data on 2 channels, the maximum sampling rate is divided by 2.
Sweep size (s) Length of the signal which is updated on the screen (sweep_len).
Buffer size (words) Size of an individual buffer (n_words)
Number of buffers Number of buffers available (n buffers)
   

Data flow
This defines mainly how long data will be stored when you decide to save them on the disk.
Continuous/Limited Select limited if you wish that the acquisition stops after a predefined time (in the current implementation, this option is the only available).
Duration (s) Time duration in seconds of the acquisition (when the option "limited" is checked).
Undersample factor 1=no undersampling; n= each data point is the average of n consecutive points and the final sampling frequency is rate / n. This operation is done "on-the-fly".
Online display Display (or not) data during the data acquisition. Unchecking this option may help to avoid loosing data if your sampling rate is too high for the computer to let it refresh the screen.
Write to disk Save data to disk.
Audible notification Emit a beep at the end of a fixed time data acquisition.
   

Trigger
These parameters define how the acquisition will start.
Software Start acquisition as soon as the button "Start" is depressed (and the experiment defined).
Keyboard Start acquisition when the user press "ok" button.
External trigger Start acquisition when the card received a positive or negative TTL signal. Look at the technical documentation of your A/D card to find the correct specifications of this signal. On Data Translation cards, there is an input channel labled as A/D TRIG which is monitored by the card and which enables the data acquisition.
   
Buttons  
OK / Cancel OK will confirm and save the current settings
Channels... Jump to the companion dialog box which defines sampling intervals
Sampling... (greyed - it refers to the companion Dialog box where the same button is active)


Notes:

In this dialog box, make sure that "software trigger" or "keyboard trigger" are selected.

If you hit OK, you return to the previous screen, if you hit "Channels...", you will return to the amplifier settings dialog box.

These parameters will be saved when you exit the program. However, if you go to "database" and return to "Record", the program will load the data acquisition parameters of the record that was selected in the database and erase your current parameters.
